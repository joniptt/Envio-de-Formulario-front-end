É uma relação entre: Cliente (Client) Garçom (Pedidos, levar seus pedidos para a cozinha e trazer) (API) Cozinha (Server) Significado de API, Application Programming Interface (Interface de programação de aplicação) é basicamente um conjunto de rotinas e padrões estabelecidos por uma aplicação, para que outras aplicações possam utilizar as funcionalidades desta aplicação. -Responsável por estabelecer comunicação entre diferentes serviços. -Meio de campo entre as tecnologias. -Intermediador para troca de informações. A transferencia de dados executada pelo API é feita atravez do protocólo HTTP. Obrigações e regras a serem seguidas na transferencia de dados feita pela API. Significado de REST, Representational State Transfer(Transferencia de Estado Representativo) ou seja será feita a transferencia de dados de uma maneira simbólica, figurativa, representativa, de maneira didática. Resources seria uma entidade, um objeto. <6 NECESSIDADES (CONSTRAITS) PARA SER RESTFUL> - _Client-Server_: Separação do cliente e do armazenamento de dados(servidor), dessa forma, é possível ter uma maior portabilidade para o nosso sistema, usando o React para WEB e o React Native para smartphones, por exemplo. - _Stateless_: Cada requisição que o cliente fizer para o servidor, deverá conter todas as informações necessárias para o servidor entender e responder (RESPONSE) a requisição (REQUEST). Exemplo: A sessão do usuário deverá ser enviada em todas as requisições, para saber se aquele usuário está autenticado e apto para usar os serviços, e o servidor não pode lembrar que o cliente foi autenticado na sessão anterior. O padrão é usar tokens para comunicação. - _Cacheable_: As respostas para uma requisição , deverão ser explicitas ao dizer se aquela requisão , pode ou não ser cacheada pelo cliente. - _Layered System_: O cliente acessa um endpoint, sem precisar saber da complexidade, de quais passos estão sendo necessários para o servidor responder a requisição, ou quais outras camadas o servidor estará lidando para que a requisição seja respondida. - _Code on Demand (optional): Da possibilidade da nossa aplicação pegar códigos como o JavaScript e executar no servidor. - Restful, é a aplicação dos padrões REST - Utilizar os verbos http para nossas requisições. - Não utilizar barras no final do ENDPOINT. - Utilizar os endpoints no plural ou no singular não importa, siga um padrão. - Nunca deixe o cliente sem resposta! - Utilizar o underline( _ ) para declarar uma parametro que não vai ser utilizado. // GET: Receber dados de um Resource. // PUT: Atualizar os dados de um Resource. // POST: Enviar dados ou informações para serem processados por um Resource. // DELETE: Deletar um resource. - 1xx: Informação - 2xx: Sucesso - 200: OK - 201: CREATED - 204: Não tem conteudo PUT POST DELETE - 3xx: Redirection - 4xx: Client ERROR - 400: Bad Request - 404: Not Found! - 5xx: Sever Error - 500: Internal Server Error
